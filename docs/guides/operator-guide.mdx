# Operator Guide

*Role: Operator* | *Last Updated: January 2025*

Welcome to the Sentia Manufacturing Dashboard! As an **Operator**, you have all Viewer capabilities plus the ability to import data, perform basic corrections, and execute retry operations. This guide covers your specific operational responsibilities.

## Your Role & Permissions

### What You Can Do

- ✅ All Viewer capabilities (dashboards, reports, exports)
- ✅ Import data from various sources
- ✅ Correct data entry errors
- ✅ Retry failed data processing jobs
- ✅ Monitor data quality and resolve issues
- ✅ Execute predefined data maintenance tasks
- ❌ Approve/reject optimization recommendations
- ❌ Modify user settings or system configuration
- ❌ Access admin functions

### File Import Process

#### Step 1: Access Import Interface
1. Navigate to **Data** → **Import**
2. Select import type (Sales, Inventory, Financial, etc.)
3. Choose data source (File Upload or Integration)

#### Step 2: File Upload
```bash
Supported formats:
- Excel: .xlsx, .xls (max 50MB)
- CSV: .csv (max 100MB) 
- JSON: .json (max 25MB)
```

1. **Drag and drop** files or **click to browse**
2. **Preview** uploaded file structure
3. **Validate** data format before processing

#### Step 3: Column Mapping

Map your file columns to system fields:

| Your Column | System Field | Required | Notes |
|-------------|--------------|----------|-------|
| Product_Code | SKU | Yes | Must be unique |
| Item_Name | Product_Name | Yes | 255 char limit |
| Stock_Qty | Quantity | Yes | Numeric only |
| Unit_Price | Price | No | Default currency |

![Column Mapping](/assets/docs/screenshots/column-mapping.png)

#### Step 4: Data Validation

The system automatically validates:
- **Required Fields**: All mandatory fields populated
- **Data Types**: Numeric, date, text validation
- **Business Rules**: Price > 0, Quantity >= 0
- **Duplicates**: Identify duplicate records

#### Step 5: Import Execution

1. Review validation summary
2. Choose handling for errors:
   - **Skip errors**: Process valid records only
   - **Fail on errors**: Stop if any validation fails
   - **Quarantine errors**: Move invalid records for review

3. Click **"Start Import"**
4. Monitor progress in real-time

## Data Quality & Error Resolution

### Error Types & Resolution

#### Validation Errors

1. **Missing Required Fields**
   - **Error**: "SKU field is required"
   - **Solution**: Add missing SKU values
   - **Tools**: Bulk edit in quarantine area

2. **Data Type Mismatches**
   - **Error**: "Price must be numeric"
   - **Solution**: Convert text to numbers
   - **Tools**: Data transformation rules

3. **Business Rule Violations**
   - **Error**: "Negative inventory quantity"
   - **Solution**: Review source data
   - **Tools**: Exception reporting

#### Data Quality Issues

1. **Duplicate Records**
   - **Detection**: Automatic duplicate identification
   - **Resolution**: Choose master record or merge
   - **Prevention**: Set up deduplication rules

2. **Inconsistent Formatting**
   - **Issue**: Date formats (MM/DD/YYYY vs DD/MM/YYYY)
   - **Solution**: Apply formatting templates
   - **Tool**: Format standardization

3. **Missing Reference Data**
   - **Issue**: Product SKU not found in master data
   - **Solution**: Create new product or map to existing
   - **Tool**: Reference data management

### Error Correction Workflows

#### Quarantine Management

Access quarantined records:
1. Go to **Data** → **Quarantine**
2. Filter by error type or import batch
3. Select records for correction
4. Apply bulk corrections or edit individually

#### Bulk Correction Tools

1. **Find & Replace**
   - Replace specific values across multiple records
   - Support for regex patterns
   - Preview changes before applying

2. **Data Transformation**
   - Apply formatting rules (uppercase, lowercase, trim)
   - Date format standardization
   - Unit conversions

3. **Reference Mapping**
   - Map unknown values to master data
   - Create mapping rules for future imports
   - Bulk approval of new reference items

## Retry Operations

### Failed Job Recovery

#### Identifying Failed Jobs
1. Navigate to **Operations** → **Job Queue**
2. Filter by status: **Failed**
3. Review error details and root cause
4. Determine if retry is appropriate

#### Retry Decision Matrix

| Error Type | Retry Action | Notes |
|------------|--------------|-------|
| Network Timeout | Automatic Retry | Usually resolves itself |
| Invalid Data | Fix Then Retry | Requires data correction |
| System Error | Manual Retry | Check system status first |
| Business Rule | Don't Retry | Fix business logic first |

#### Manual Retry Process

1. **Select Failed Job**: Click on failed job in queue
2. **Review Error**: Understand failure reason
3. **Fix Issues**: Resolve underlying problems
4. **Retry Job**: Click **"Retry"** button
5. **Monitor Progress**: Watch for successful completion

![Job Retry](/assets/docs/screenshots/job-retry.png)

### Batch Retry Operations

For multiple failed jobs:
1. **Select Multiple**: Use checkboxes to select jobs
2. **Bulk Actions**: Choose **"Retry Selected"**
3. **Confirmation**: Review jobs to be retried
4. **Execute**: Start batch retry process

## Troubleshooting Guide

### Common Import Issues

#### "File Format Not Recognized"
**Symptoms**: Upload fails with format error
**Solutions**:
1. Verify file extension matches content
2. Check for file corruption
3. Try saving in different format
4. Remove special characters from filename

#### "Column Mapping Failed"
**Symptoms**: Cannot map required fields
**Solutions**:
1. Check column headers match expected format
2. Verify data types in source file
3. Remove empty columns
4. Check for merged cells in Excel

#### "Import Stuck in Processing"
**Symptoms**: Import shows processing for >30 minutes
**Solutions**:
1. Check system status dashboard
2. Contact administrator if system issue
3. Cancel and restart import if possible
4. Try smaller batch size

### Integration Issues

#### "Shopify Connection Lost"
**Symptoms**: Sync fails with authentication error
**Solutions**:
1. Check API credentials in settings
2. Verify Shopify app permissions
3. Re-authenticate if needed
4. Contact administrator for credential updates

#### "Amazon API Rate Limit"
**Symptoms**: Sync fails with rate limit error
**Solutions**:
1. Wait for rate limit reset (usually 1 hour)
2. Reduce sync frequency temporarily
3. Contact administrator to review API usage
